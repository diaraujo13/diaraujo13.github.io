Glossário

Termos e definições explicadas em linguagem simples.

ABCDEFGHIJKLMNOPQRSTUVWXYZ

A

A/B Test

Experimento para comparar duas versões (A e B) e medir qual performa melhor.

Algoritmo

Sequência de passos lógicos para resolver um problema.

Android

Sistema operacional móvel do Google.

API

Interface de Programação de Aplicações; forma padronizada de sistemas conversarem entre si.

App Store

Loja de aplicativos da Apple (iOS).

Artificial Intelligence (IA)

Área de computação que desenvolve sistemas capazes de realizar tarefas que exigem inteligência humana.

B

Back-end

Parte do sistema que roda no servidor (lógica, APIs, bancos).

Banco de Dados

Local estruturado para armazenar e consultar informações.

Banco NoSQL

Banco não relacional (ex.: MongoDB) focado em flexibilidade e escala.

Banco SQL

Banco relacional (ex.: PostgreSQL, MySQL) com tabelas e relações.

Binário

Formato de dados em 0 e 1 que o computador entende.

C

CD (Continuous Deployment)

Implantação Contínua; publicar automaticamente novas versões em produção após passar por etapas de qualidade.

CDN

Rede de distribuição de conteúdo para entregar arquivos rapidamente.

CI (Continuous Integration)

Integração Contínua; prática de integrar e validar mudanças de código com frequência automática.

Commit

Registro de uma mudança de código em um repositório (ex.: Git).

D

Dependências

Bibliotecas/peças de software que seu projeto utiliza.

Dependências (NPM/Yarn)

Pacotes reutilizáveis que aceleram o desenvolvimento.

Deploy

Ato de publicar/atualizar o sistema em um ambiente (homologação/produção).

DigitalOcean

Provedor de nuvem para hospedar servidores, bancos, etc.

Docker

Tecnologia de contêiner para empacotar e rodar apps de forma isolada.

Domínio

Endereço do seu site na internet (ex.: exemplo.com).

E

Engenharia de Requisitos

Disciplina que define o que o software deve fazer (requisitos).

Engenharia de Software

Área que estuda métodos e práticas para criar software de qualidade.

F

Firebase

Plataforma do Google com serviços para apps (auth, banco, push, analytics).

Front-end

Parte do sistema que o usuário vê e interage (interface).

G

Gateway de Pagamento

É um serviço que permite que os usuários realizem pagamentos online.

GitHub

Plataforma para hospedar código (Git) e colaborar.

Google Analytics

Ferramenta do Google para medir tráfego e uso do site.

Google Play

Loja de aplicativos do Android.

H

Heroku

Plataforma (PaaS) para deploy de apps sem gerenciar servidores.

Homologação

Ambiente de testes finais antes de ir à produção.

HTTP

Protocolo usado para comunicação na web (requisições/respostas).

I

i18n (Internacionalização)

Preparar o app para múltiplos idiomas e formatos.

iOS

Sistema operacional móvel da Apple.

K

KMM (Kotlin Multiplatform)

Estratégia para compartilhar código Kotlin entre Android, iOS e backend.

Kotlin

Linguagem moderna para Android e back-end.

Ktor

Framework Kotlin para back-ends e APIs.

Kubernetes

Orquestrador de contêineres para escalar e gerenciar serviços.

L

Latência

Tempo de ida e volta de uma requisição até a resposta.

Linux

É um sistema operacional livre e de código aberto, conhecido por ser um sistema operacional de alto desempenho e segurança.

M

Multi-tenant

É um sistema que permite que múltiplos clientes compartilhem o mesmo sistema, mas com isolamento de dados.

N

Navegador

Programa para acessar a web (Chrome, Safari, Firefox).

Nginx

Servidor web/reverso de alta performance.

NGINX

Variação de escrita comum para Nginx.

Node.js

Runtime JavaScript no servidor.

NPM

Gerenciador de pacotes do ecossistema Node.js.

O

Objetive-C

Linguagem usada historicamente no iOS/macOS (antes do Swift).

Operating System (SO)

Sistema Operacional; software que gerencia o hardware (Windows, Linux, iOS, Android).

P

PaaS (Platform as a Service)

Modelo de nuvem que entrega plataforma pronta para rodar apps (ex.: Heroku).

Porta

Canal lógico de comunicação em um servidor (ex.: 80, 443).

Processo de Desenvolvimento

Sequência de etapas para realizar uma tarefa.

Produção

Ambiente real com usuários de verdade.

Protocolo

Conjunto de regras para dois sistemas se comunicarem (ex.: HTTP, SMTP).

Push Notification

Mensagem enviada ao dispositivo do usuário mesmo quando o app está fechado.

R

React

Biblioteca JavaScript para construir interfaces (front-end).

React Native

Framework para criar apps nativos usando React e JavaScript.

Requisitos Funcionais

O que o sistema deve fazer (funcionalidades).

Requisitos Não Funcionais

Como o sistema deve ser (segurança, desempenho, usabilidade).

S

SaaS (Software as a Service)

Software entregue como serviço pela internet, por assinatura.

SCRUM

Metodologia ágil com sprints, papéis e eventos para organizar o trabalho.

Serviço de E-mail

Servidor/serviço responsável por enviar e receber e-mails (SMTP).

Servidor de Aplicação

Programa que executa o código do sistema no servidor.

SMTP

Protocolo para envio de e-mails.

Splash Screen

Tela inicial exibida ao abrir um app, enquanto carrega.

Swift

Linguagem moderna da Apple para iOS/macOS.

SwiftUI

Framework de UI declarativa para iOS/macOS.

T

Testes Caixa Branca

Testes que conhecem a implementação interna do código.

Testes Caixa Preta

Testes que avaliam entradas e saídas sem conhecer o código.

Testes de Aceitação

Validam se o sistema atende ao que foi combinado com o cliente.

Thread

Unidade de execução dentro de um processo; permite paralelismo/concorrência.

Tráfego Pago

Anúncios pagos para atrair visitantes (Google, Meta, etc.).

U

UI (User Interface)

Camada visual do produto (botões, cores, tipografia).

UX (User Experience)

Experiência do usuário ao usar o produto (fluidez, clareza, satisfação).

V

Versionamento

Controle de versões do código (Git).

VPN

Rede privada virtual; túnel seguro pela internet.

Y

Yarn

Gerenciador de pacotes alternativo ao NPM.
